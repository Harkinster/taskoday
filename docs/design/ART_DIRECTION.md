# Taskoday Art Direction

## Identity

Taskoday is an Android Jetpack Compose app with a fantasy productivity skin. The new gamification layer should feel like a small magical sanctuary, not a separate game engine.

Official vocabulary:

- Le Nid: main progression hub.
- Gardien: progression identity for the child.
- XP du Gardien: progression only, never spent.
- Ecailles: external reward currency only.
- Coffres: loot containers earned by task completion.
- Loot items: materials used for eggs and dragon evolution.
- Oeufs: hatchable collection items.
- Dragons: companions unlocked by hatching.
- Caverne aux Souhaits: parent-created real rewards.
- Souhait: child request for a real reward.
- Parchemin: approved real reward coupon.

## Visual Rules

- Keep the existing Taskoday fantasy UI kit: dark blue surfaces, neon cyan/violet highlights, compact cards, and avatar access to profile.
- Do not introduce a game engine, Unity-like HUD, or heavy animated scenes for the MVP.
- The Nid should be the first app signal after login.
- Real rewards must feel separate from progression: use Ecailles copy and avoid XP wording in the Caverne aux Souhaits.
- Dragons and eggs should use warm magical item cards until final raster assets are available.

## Screen Tone

- Le Nid: calm, encouraging, progression-focused.
- Inventory: readable list of items and quantities.
- Oeufs: incubator-like, clear requirements.
- Dragons: collection and evolution state.
- Caverne aux Souhaits: parent/child transaction surface, restrained and explicit.
- Parchemins: simple redemption tracking.
